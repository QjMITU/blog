Mybatis源码学习之配置文件解析

前言

作为一个Java开发者，从Servlet+JDBC，到如今的SpringBoot+Mybatis等框架，开发系统CRUD功能的难度以及代码量也是越来越低了。因此Java开发的门槛也是比较低的，通过海量培训机构的教程花费几个月的时间学习框架，就能处理基础的工作内容了。但是随着竞争压力的越来越大，基础的技能素养已经不再具备任何竞争力了，因此为了提高自身的能力，并且方便后续回顾，这里在阅读Mybatis源码之后，进行相关流程的分析记录。

本章将主要介绍Mybatis的mybatis-config.xml配置文件的解析过程。需要源码的请自己从GitHub拉取，这里用的是官方32版本。

入口main方法

main方法的前2行，主要是获取配置文件的stream流对象，后面将在==SqlSessionFactory sessionFactory = new SqlSessionFactoryBuilder().build(stream);== 中解析。

这里对mybatis-config.xml中的内容先忽略，后面在解析的源码出可以清晰看到。

new SqlSessionFactoryBuilder().build(stream)

SqlSessionFactoryBuilder这里用了建造者模式，build方法有甚多重载，但都是大同小异，不同的地方是获取配置文件的方式，可以用stream也可以用reader，在build方法中，处理了大部分逻辑，如解析mybatis-config.xml，解析XXXmapper.xml，注册别名等，最后会返回一个SqlSessionFactory接口用来创建SqlSession，这里用了抽象工厂模式，实际创建的是==DefaultSqlSessionFactory==和==DefaultSqlSession==。

build(stream)

build方法，解析inputStream也就是mybatis-config.xml内容，将解析的内容存入Configuration中，然后调用另一个重载方法返回一个DefaultSqlSessionFactory。

XMLConfigBuilder的parse()方法

这个方法进来先判断了一下有没有解析过，这里奇怪，莫非是为了防止有人保存这个对象引用重复调用吗？

==parseConfiguration(parser.evalNode(“/configuration”));==主要是这里，获取configuration标签的内容然后解析。

parseConfiguration(parser.evalNode(“/configuration”));

从这个方法可以看出，configuration标签中主要有哪些内容：

  • properties
  • typeAliases
  • plugins
  • objectFactory
  • objectWrapperFactory
  • settings
  • environments
  • databaseIdProvider
  • typeHandlers
  • mappers

总结

看到这里，后面就开始分步进行配置文件的解析了然后我们总结一下XMLConfigBuilder类。该类中parsed用来标记文件是否被解析过，parser是xml文件的Java对象，包含了mybatis-config.xml的数据，environment是null，另外再其父类中，有一个configuration，包含了所有解析出来的配置，另外两个注册器，也是从configuration中获取的。
